How to Download Older Versions


Instructions to download selected version "version name here"

For other game versions: select desired game version in Version of Save selector above, or refer to the Reference Table section below.

Note: with file names and commands, DO NOT include any quotation marks ("") surrounding them.

  • Step 1: Steam Console command

    First, open the Steam Console.

    • On Windows: Open the Run dialog box with  + R,
      then enter the command "steam://open/console". Click Ok.
    • On MacOS: Open a browser and enter "steam://open/console" in the address bar at the top.
      (Then hit Enter.)
    • On Linux: Open a browser and enter "steam://open/console" in the address bar at the top.
      (Then hit Enter.)

    This should open Steam with a "Console" tab showing.

  • Step 2: Download the depot manifest using Steam Console

    Inside the steam console, enter the following command depending on your operating system:

    • On Windows: "download_depot 1451480 DEPOT_ID MANIFEST_ID"
    • On MacOS: "download_depot 1451480 DEPOT_ID MANIFEST_ID"
    • On Linux: "download_depot 1451480 DEPOT_ID MANIFEST_ID"

    You should see a message like "Downloading Depot 1451481 ..." appear. This means it is downloading. (Depot ID may vary!)
    You may have to wait a while; most versions of this game are around 1-2 GB compressed.

    • What if I saw an error instead? Such as "Depot download failed : ..."?

      First, check that your Depot ID and Manifest ID are correct! That can be done by checking the Reference Table section below, or looking it up in the SteamDB depots for this game.
      Note: For the same game version there exist three different Manifest IDs, one from each Depot (desired operating system). Check that your Manifest ID and Depot ID are for the same operating system.

      If they are correct and you still receive an error, you can try using the well-known third-party tool DepotDownloader. I recommend this video as a tutorial. It was for the Hollow Knight game community, so be sure to use your App ID, Depot ID, and Manifest ID in place of theirs in the video.

  • Step 3: Locate and Move Download

    After a successful download you should see a message that looks something like:
    Depot download complete : "<Path to download folder location>" (<...> files, manifest <Manifest ID>)
    Depot ID and manifest will vary depending on your operating system and version selected. The path will often end in something like ".../app_1451480/depot_1451481".
    If you see errors afterwards, such as "LogonFailure No Connection", don't worry. After "Depot download complete" you already have the files you needed.

    Go to the download folder location, and you should see contents in that folder structured something like this:

    • <download folder>
      ├── MonoBleedingEdge
      ├── Penguin Heist_Data
      ├── Penguin Heist.exe
      ├── UnityCrashHandler64.exe
      └── UnityPlayer.dll

    • <download folder>
         └── Penguin Heist.app
            └── Contents
               ├── _CodeSignature
               ├── Frameworks
               ├── MacOS
               │   └── Penguin Heist
               ├── MonoBleedingEdge
               ├── Plugins
               ├── Resources
               └── Info.plist

    • <download folder>
      ├── PenguinHeist_Data
      ├── PenguinHeist
      ├── PenguinHeist_s.debug
      ├── UnityPlayer.so
      └── UnityPlayer_s.debug

    Depending on the game version you download, the contents may look slightly different.

    Those are the files needed to run this copy of the game! Let's move these for safety, so Steam doesn't overwrite them.

    1. Create another folder somewhere (such as your Desktop or Documents).
    2. Copy or move all the files from the download folder to the new folder.

    Anytime you want to run the game, just run the game executable file (Penguin Heist.exe, Penguin Heist, or PenguinHeist).

    • Congratulations! Anytime you want to play this version, you can run this game executable!

    IMPORTANT: See the reminder below at the bottom of these instructions about where the game saves its data! No matter which game version or where the game is, it will always try to read and write the save file to the same location.

  • Step 4 (Optional): Steam integration for visibility achievements and game hours

    Currently, Steam will not know when this copy of the game is running. If you want Steam to show when you're playing Penguin Heist, keep track of achievements, count game hours, etc., follow this step.

    In the folder containing the game executable file (Penguin Heist.exe, PenguinHeist, or PenguinHeist), create a new text file (using Notepad or textedit or any other), and put in the digits 1451480, which is this game's Steam App ID. Nothing else!
    Save the text file as "steam_appid.txt" and put that text file into your game's folder, ie., next to the executable file.

  • Step 5 (Optional): Add to Steam as non-Steam game

    If you want to run this copy of the game from Steam just like you run the original one, You can add the game executable to Steam as a non-Steam game.
    Open the Steam client window and select the "Add a Game" on the bottom left > "Add a non-Steam Game" > "Browse", and navigate to the game executable file (Penguin Heist.exe, Penguin Heist, or PenguinHeist). Select it in the file browser. Ensure the game executable is checked and click "Add Selected Programs".
    Now you can find it in your Library with the same name as the executable file. Rename or organize it however you like in Steam.

  • Remember: Game save file location is the same!

    No matter which game version or where the game is, it will always try to read and write the save file to the same location!
    Keep in mind: playing different versions using the same save file can cause (often small) side effects. The game tends to ignore properties that it does not recognize. For example, before the Blueprint Update, shop items were restricted with a constructions and research system. The Blueprint Update outmoded this constructions system. Save files that came from before to after this change simply switched over to the new system, but the shop restricted purchases based on blueprint collection instead.

    If you are okay with small side effects, no worries!

    However, if you ever want to keep separate saves (even for the same game version), I would recommend renaming the save folder "FishSoft" to something else, and then the game will leave that save alone.
    When you want to switch back, simply rename the current save folder and rename the desired one back to "FishSoft".

    • If there's no save folder with the expected name ("FishSoft") then the game will create a completely fresh save file and folder, the default for that game version (ie., no extra unrecognized properties.)

    Examples of side effects:

    • Older example: If you take a save file generated on a newer version (with items unlocked via collecting the blueprints) and then play that save file with an old game version before the Blueprint Update, the shop would restrict what you can purchase due to needing blueprints now instead of the old constructions system. However, any items you had already purchased previously would still be available for your loadout, even if they got locked in the shop.
    • Newer example: Patch + Cosmetics + Heist Planner Update added dozens of new cosmetic items. If you equip your player with any of those new cosmetic items, and then play the game in an older game version, it may forcibly un-equip clothing from your "equipped" clothing list. Then you would need to re-equip those clothings again if you choose to play it in the newer version. You would still have ownership of purchased new clothings; those are separate save file properties. The older version just doesn't know the clothing, and therefore you couldn't access them.

    And no matter what happens, you can use this save editor tool to inspect (and edit) exactly what properties the save file has! :)

If these instructions are incorrect, please let me know what's wrong or didn't work by submitting a GitHub issue here or @ me on Discord.
